概述

WxAuto HTTP API 是基于 wxauto/wxautox 库开发的微信自动化HTTP接口服务，提供简单易用的RESTful API来控制微信客户端。
项目关系说明

wxauto-http-api 是基于 wxauto/wxautox 进行二次开发的HTTP接口项目，本人非wxauto原作者，不销售wxautox！

主要特性

  • 🚀 简单易用 - RESTful API设计，支持多种编程语言
  • 🔒 安全可靠 - API密钥认证，确保接口安全
  • 📱 功能丰富 - 支持消息发送、文件传输、群组管理等
  • 🔄 实时监听 - 支持消息监听和实时获取
  • 🎯 精确控制 - 基于UI自动化，操作精确可靠
  • 📊 状态监控 - 实时监控微信状态和连接情况

技术架构

  • 后端框架: Flask + Python
  • 微信控制: wxauto/wxautox
  • API风格: RESTful
  • 认证方式: API Key
  • 数据格式: JSON
  • 平台支持: Windows

快速开始

启动服务

运行HTTP API服务，默认端口5000

python app.py
初始化微信

调用初始化接口连接微信客户端

POST /api/wechat/initialize
发送消息

使用Chat类API发送消息

POST /api/chat/send-message

常见问题

支持微信3.9.8及以上版本，推荐使用3.9.8版本以获得最佳稳定性。

本项目基于Windows官方API开发，不涉及任何侵入、破解、抓包微信客户端应用，完全以人操作微信的行为执行操作，正常使用不会被封号。

API密钥在配置文件 data/api/config/app_config.json 中设置，默认为 test-key-2。

  • wxauto：开源免费使用，满足日常消息收发需求
  • wxautox：如果有复杂需求请联系原作者进行购买wxautox
  • 注意：本项目为基于wxauto/x进行二次开发，本人非原作者！

如果出现"另一个UI实例已经启动，请不要重复启动"错误：

  1. 按 Win + R 打开运行对话框
  2. 输入 %temp% 并按回车
  3. 在打开的临时文件夹中找到并删除 wxauto_http_api_ui.lock 文件
  4. 重新启动程序
重要提示：
  • 请确保微信客户端已登录并保持在前台
  • 首次使用需要调用初始化接口
  • 建议在测试环境中先熟悉各个接口的使用方法
  • 生产环境使用时请修改默认的API密钥
